生前整理サービス 画面モック一覧

各画面のHTMLモック。Claude Codeで実装する際の出発点として参照する。

画面の役割と関係

画面A：ログイン　最初の入口。Googleアカウントでログイン。

画面C：ダッシュボード　ホーム画面。全体進捗、4グループの状況、次にやることのサジェスト。

画面D：明細アップロード　明細PDF/CSVをアップロード（Google Driveに保存）。

画面E：明細抽出結果の確認　Claude Codeで解析された結果を、家族が確認・登録。

画面F：項目詳細（4観点）　中核画面。1項目に対して、4観点を2x2のグリッドで表示・編集。

画面G：項目追加フロー　新規項目の追加（グループ→項目タイプ→入力）。

画面H：項目一覧　項目を縦に並べる。詳細表示と簡易表示の切り替え可能。

画面I：TODO一覧　整えるべきタスクの一覧。

UI設計の共通方針

カード型UI：項目を白地のカードで表示。シャドウで浮き上がる感じ。

色の意味：緑=完了、黄=未確認、グレー=該当なし、青=操作可能なボタンや情報

4観点の表示：項目詳細では2x2のグリッド、一覧の簡易表示では小さなアイコン4つ

レスポンシブ：PC・タブレット・スマホで使えるように。Tailwindのレスポンシブクラスで対応。

フォント・サイズ：基本は14px、見出しは16-22px。余白を多めに取って読みやすく。

デザイントーンについて

このモックは、構造を確認するための仮置きデザイン。
実装時に、寺内さんが複数のトーン案（自然系/信頼系/温かさ系など）から判断する。
LP（tlife.info）とは別のブランディングが必要（LPは社外財務サポーター用）。

Claude Codeでの使い方

1. 各モックHTMLを開いて構造を確認

2. 実装する画面のモックを参照して、コードを生成

3. データはGoogleスプシから取得、スプシのカラム構成は設計ドキュメント参照

4. Tailwind CSSを使用、CDNから読み込み（モックと同じ構成）

5. 認証はGoogle OAuth、APIは Google Sheets API / Drive API
